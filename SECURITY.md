# Security Policy

## Supported Versions

| Version | Supported          |
|---------|--------------------|
| Latest (main) | :white_check_mark: |
| Older versions | :x:                |

## Reporting a Vulnerability

**Please do NOT report security vulnerabilities through public GitHub issues.**

We take security seriously. If you discover a security issue, please report it privately.

### How to Report

- **Preferred method**: Use [GitHub Private Vulnerability Reporting](https://github.com/KADHIRAVANEG/cloudcompliance/security/advisories) (if enabled)
- **Alternative**: Email us at `security@yourdomain.com` or `[your-github-username]@users.noreply.github.com`

### What to Include in Your Report

- Description of the vulnerability
- Steps to reproduce the issue
- Affected versions / components
- Potential impact (e.g. data leak, RCE, etc.)
- Any suggested fixes or mitigations (if you have them)

### Response Timeline

- **Acknowledgment**: Within 48 hours
- **Detailed update**: Within 5 business days
- We will work with you to fix and disclose responsibly

## Disclosure Policy

- We follow [responsible disclosure](https://en.wikipedia.org/wiki/Responsible_disclosure).
- We will credit you for the discovery (unless you prefer to stay anonymous).
- We aim to release a fix as soon as possible after validation.

---

Thank you for helping keep this project secure! 🙏
